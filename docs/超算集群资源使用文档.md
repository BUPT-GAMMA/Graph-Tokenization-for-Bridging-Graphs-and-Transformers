## 8.4 超参数搜索策略

### 8.4.1 方案A：批量作业提交

创建超参数搜索脚本 `submit_hyperparameter_jobs.py`：
```python
#!/usr/bin/env python3
import subprocess
import itertools

# 定义超参数空间
learning_rates = [1e-4, 5e-4, 1e-3]
batch_sizes = [16, 32, 64]
bpe_merges = [1000, 2000, 5000]

for lr, bs, merges in itertools.product(learning_rates, batch_sizes, bpe_merges):
    job_name = f"hyper_lr{lr}_bs{bs}_merge{merges}"
    job_script = f"""#!/bin/bash
#SBATCH -J {job_name}
#SBATCH -N 1
#SBATCH -p a01
#SBATCH -o stdout_{job_name}.%j
#SBATCH -e stderr_{job_name}.%j
#SBATCH --no-requeue
#SBATCH --ntasks-per-node=4
#SBATCH --gres=gpu:1

source /apps/soft/anaconda3/bin/activate
conda activate your_env

cd $WORK/tokenizerGraph
python bert_finetuning_pipeline_normalized.py --learning_rate {lr} --batch_size {bs} --bpe_num_merges {merges}
"""
    
    with open(f"job_{job_name}.sh", "w") as f:
        f.write(job_script)
    
    subprocess.run(["sbatch", f"job_{job_name}.sh"])
```

## 8.7 常见问题处理

### 8.7.1 作业异常状态处理
- **PD状态长时间等待**：检查是否达到资源限制，考虑调整资源需求
- **CG状态（异常结束）**：查看stderr文件，检查代码错误或资源不足
- **节点故障**：在脚本中添加`#SBATCH -x 故障节点名`排除故障节点

### 8.7.2 网络访问问题
- **包安装失败**：确保在GPU计算节点上安装，使用清华镜像源
- **Hugging Face访问**：使用hf-mirror.com替代huggingface.co

### 8.7.3 路径和权限问题
- **数据路径错误**：确保所有路径都指向$WORK目录
- **权限问题**：检查文件权限，避免在不当位置创建文件

这样既能验证环境配置，又能熟悉集群的作业调度系统，为后续大规模实验做好准备。
